## Merge Report Contract

Canonical type in `packages/core/src/index.ts:25` (`MergeReport`):

- `kind: 'drawings' | 'specs'`
- `originalPath: string`
- `addendumPaths: string[]`
- `outputPath?: string`
- `replaced[]`
- `inserted[]`
- `appendedUnmatched[]`
- `warnings: string[]`
- `notices?: string[]`
- `stats: { originalPages, finalPagesPlanned, parseTimeMs, mergeTimeMs }`

Generated in `packages/core/src/core/report.ts`.

Notice/warning split behavior:

- expected cover-page ROI failures are moved to `notices`
- actionable detection/merge concerns stay in `warnings`

This distinction is intentional and should be preserved.

## Inventory File Contract

Inventory JSON write path in `packages/core/src/core/planner.ts`:

- default output filename: `{sourcePdfBase}-inventory.json`
- location:
  - default next to source PDF
  - or `inventoryOutputDir` override

Top-level inventory fields:

- `sourceFile`
- `type`
- `totalPages`
- `parseTimeMs`
- `pagesWithIds`
- `pagesWithoutIds`
- `warnings` (count)
- `inventory` (per-page array)
- `performance` (parse timing metrics)

Per-page inventory rows include context fields such as:

- `pageIndex`
- `sheetId` or `sectionId`
- normalized IDs
- title
- confidence
- source/method
- context/warning

## Detection/Preview Result Contract

Detection result shape in `conset-pdf-gui/src/main/utils/detection-orchestration.ts:21` (`PageDetectionResult`):

- `pageIndex`, `pageNumber`
- `sheetId` object with found/value/normalized/confidence/failure metadata
- optional `sheetTitle`
- optional `warnings`, `context`

This is the effective preview wire shape used by GUI detection workflows.

## Naming Convention Contract

From `docs/OUTPUT_STRUCTURE.md` and planner/report implementation:

- inventory: `{source}-inventory.json`
- report: user-specified path (recommended reports directory)
- preview: user-specified or `{source}-preview-{timestamp}.json` patterns
- merged PDF: user-defined output filename

## Rust Mapping Notes

- Preserve key names exactly for report/inventory JSON compatibility.
- Keep unmatched reason enum strings (`no-id`, `ambiguous`, `unmatched`).
- Preserve notices vs warnings separation and not just a single diagnostics list.
- Ensure deterministic ordering for arrays that are compared in regression tests.
