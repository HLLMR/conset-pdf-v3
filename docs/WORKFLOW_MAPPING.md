# Workflow Engine Mapping Notes

**Last verified**: 2026-01-17

This document explains how existing merge structures map to the new workflow engine types.

## MergePlan → InventoryResult.summary

The `MergePlan` structure contains:
- `pages`: Array of final plan pages
- `replaced`: Array of replacement operations
- `inserted`: Array of insertion operations
- `unmatched`: Array of unmatched pages
- `parseWarnings`: Array of warning strings

**Mapping:**
- `totalRows` = `mergePlan.pages.length`
- `rowsWithIds` = count of pages with `id !== undefined`
- `rowsWithoutIds` = `totalRows - rowsWithIds`
- `replaced` = `mergePlan.replaced.length` (workflow-specific field)
- `inserted` = `mergePlan.inserted.length` (workflow-specific field)
- `unmatched` = `mergePlan.unmatched.length` (workflow-specific field)
- `issuesCount` = `mergePlan.parseWarnings.length`

## ParseResult.inventory → InventoryResult.rows

The `ParseResult.inventory` structure contains per-page detection results:
- `pageIndex`: 0-based page index
- `sheetId`: Detected sheet ID (raw)
- `normalizedId`: Normalized sheet ID
- `title`: Sheet title (if available)
- `confidence`: Detection confidence (0..1)
- `source`: Detection method/source
- `context`: Additional context
- `warning`: Warning message (if any)

**Mapping:**
- `id` = Stable UID: `${pdfId}-${detectionSource}:${pageIndex}:${(sheetId||normalizedId||'')}` (does not change when normalizedId is overridden)
- `normalizedId` = `normalizedId || sheetId || undefined` (stored separately, can be overridden via corrections)
- `page` = `pageIndex + 1` (convert to 1-based)
- `status` = determined by presence of ID and warnings:
  - `'error'` if no ID found
  - `'warning'` if warning present
  - `'ok'` if ID found and no warning
- `confidence` = `confidence ?? 0`
- `source` = `source` (detection method)
- `notes` = `warning || context`
- `discipline` = added for `docType === 'drawings'` (see STANDARDS.md)
- `specs` = added for `docType === 'specs'` (see STANDARDS.md)

## MergeReport.warnings → InventoryResult.issues

The `MergeReport.warnings` is an array of warning strings.

**Mapping:**
- Each warning string becomes an `Issue` object
- `severity` = determined heuristically from warning content:
  - `'error'` if contains "error" or "failed"
  - `'info'` if contains "notice" or "info"
  - `'warning'` otherwise
- `code` = extracted from warning content (e.g., "NO_ID", "LOW_CONFIDENCE", "DUPLICATE")
- `message` = the warning string itself
- `rowIds` = empty array (could be enhanced to extract row IDs from warning)

## MergePlan.unmatched → InventoryResult.issues

The `MergePlan.unmatched` array contains:
- `reason`: 'no-id' | 'ambiguous' | 'unmatched'
- `addendumSource`: Source file path
- `pageIndexes`: Array of 0-based page indices

**Mapping:**
- Each unmatched item becomes an `Issue` object
- `severity` = 'error' for 'no-id', 'warning' for others
- `code` = 'NO_ID' | 'AMBIGUOUS' | 'UNMATCHED' based on reason
- `message` = descriptive message with page list
- `rowIds` = page indices converted to row IDs (`page-{index}`)
- `details` = original unmatched item data

## Future Enhancements

- **Conflicts**: When narrative parsing is implemented, conflicts will be generated by comparing narrative instructions to detected inventory
- **Row ID extraction**: Enhance warning parsing to extract specific row IDs from warning messages
- **Status refinement**: Cross-reference inventory rows with plan operations to determine more accurate status (e.g., rows that will be replaced should have status 'warning' or specific action)
